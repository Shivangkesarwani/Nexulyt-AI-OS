# Contributing Guidelines

Thank you for contributing to the **Nexulyt-AI-OS** repository! To ensure all additions remain production-grade, please follow these guidelines.

---

## 1. Submitting a New Skill

Every new skill must follow the standard repository layout:
1. Create a subfolder under `skills/` using `kebab-case` naming conventions.
2. Formulate `SKILL.md` (metadata, core paradigms, identity definition, behavioral rules).
3. Formulate `README.md` (overview, responsibilities, compatible skills, expected outputs).
4. Formulate `CHECKLIST.md` (detailed checkboxes mapping to standard checklists).
5. Formulate `EXAMPLES.md` (at least 3 real-world scenarios showing the thinking and decision process).

---

## 2. Review Process

- All pull requests are evaluated against `standards/engineering-rules.md` and `standards/coding-style.md`.
- No PR can be merged without passing the `Code Reviewer` validation gates.
