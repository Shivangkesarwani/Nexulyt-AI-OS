# Nexulyt-AI-OS Assets & Branding Library

Welcome to the **Nexulyt-AI-OS Assets & Branding Library**. This directory contains the guidelines, design system specifications, and layouts for the visual identity and branding assets of the Nexulyt-AI-OS repository.

---

## 1. Purpose

The purpose of this assets library is to document the repository's brand identity, color palettes, typography scales, logo safe areas, screenshots layout guides, and diagrams style. By establishing clear guidelines, we ensure that:
1. **Visual Consistency**: All visual materials (e.g. banners, logos, diagrams, user interfaces, documentation) follow a unified design system.
2. **Accessible Branding**: Color choices, contrast bounds, and typography scales conform to WCAG 2.1 AA accessibility guidelines.
3. **Structured Repository Presentation**: Visual overlays and preview cards look professional and render correctly across platform dashboards (like GitHub, npm, or Discord).

---

## 2. Repository Branding Overview

The brand identity of **Nexulyt-AI-OS** is designed around three main pillars:
* **Agentic Power**: Reflecting autonomous, reliable, and intelligent code operations.
* **Premium Precision**: Expressed via geometric structures, clean lines, high information density, and sleek dark modes.
* **Minimalist Aesthetics**: Removing visual clutter, utilizing functional styling tokens, and prioritizing clarity.

## 4. Naming & Storage Standards

To prevent folder clutter and ensure assets remain easy to manage:
1. **File Naming**: Always use lowercase kebab-case naming structures (e.g. `logo-light-mode.svg`, `dashboard-desktop-view.png`).
2. **Binary Assets Storage**: All raw binary assets (PNG, SVG, JPEG) must be stored inside the target folders declared in [Asset Organization Guide](file:///d:/projects/Nexulyt-AI-OS/assets/asset-organization.md).
3. **Compression Rule**: Image assets must be run through optimization scripts before git commits to keep repository download size minimal.
