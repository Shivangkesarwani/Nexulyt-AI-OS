# UI/UX Designer AI Skill

A production-grade AI Skill designed to teach AI assistants to think like Senior Product Designers, creating user interfaces and user experiences that look custom-designed by humans.

---

## 1. Overview
The UI/UX Designer skill defines a comprehensive design paradigm, cognitive guardrails, and aesthetic models. It prevents AI from producing generic layouts and ensures every pixel, layout grid, typographic choice, and interaction pattern has a clear, goal-oriented purpose.

---

## 2. Purpose
- **Eliminate AI Footprints:** Avoid standard, pre-made layouts, generic colors, and visual clutter.
- **HCI Integration:** Enforce layout designs based on verified Human-Computer Interaction (HCI) principles (e.g., Fitts's Law, Hick's Law, Gestalt Principles).
- **Quality & Consistency:** Enforce strict mathematical spacing grids, typography hierarchies, contrast checks, and motion design parameters.

---

## 3. Features
- **Design Signatures:** Cognitive styles inspired by Apple, Stripe, Linear, Vercel, Notion, Framer, OpenAI, and Airbnb.
- **Visual & Layout Systems:** Proportional typography scales (Major Third), HSL color palettes, 8pt spacing systems, and parent container queries.
- **Specialized Interfaces:** Layout design strategies for Portfolios, SaaS Dashboards, AI SaaS, Restaurants, Agencies, E-commerce, and Admin Panels.
- **Self Review Engine:** An automated design critique loop that evaluates simplicity, contrast, legibility, and responsive flow.

---

## 4. When to Use
Use this skill before starting any frontend coding, visual mockup, or system design stage to align user interfaces with target goals.

---

## 5. Workflow
1. **Analyze Context:** Identify target user demographics, psychology profiles, and devices.
2. **Establish Design Tokens:** Choose typography Scales, HSL colors, and spacing structures.
3. **Draft Layouts:** Place components using Flexbox/CSS Grid containers.
4. **Audit and Polish:** Run the self-criticism engine and accessibility checks before outputting styles.

---

## 6. Expected Output
When activated, this skill generates:
- High-fidelity visual mockups using semantic CSS variables.
- Clear structural layouts built using container queries and modern spacing grids.
- Real-time input validations and accessibility features (WCAG 2.2 compliant).

## 10. Best Practices
- **Define Tokens First:** Establish primitive, semantic, and component tokens before generating layouts.
- **Respect Spatial Grid:** Keep all margins, paddings, and alignment gaps within the 8pt spacing scale.
- **Verify Contrast:** Audit text color contrast using WCAG 2.2 AA standards before finalize layouts.
